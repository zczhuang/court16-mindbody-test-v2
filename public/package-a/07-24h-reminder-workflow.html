24-hour class reminder workflow

A HubSpot workflow that emails the parent the day before their scheduled trial — class time, location, and a short "what to bring" nudge.

What
Scope from the SOW: a HubSpot workflow "triggered on Scheduled Trial + class date - 24h" that sends the parent a reminder email.
Why
Reducing no-show rates without adding staff work. The reminder is a classic operational touch that pays back every time.
Who
Ibtissam builds the workflow. Template copy drafted in spec 08.
Status
Greenfield · delay + branching + copy decided at kickoff

What's defined (per the SOW)

  • Trigger: Deal is in pipeline Trials and stage Scheduled Trial — matches the SOW phrasing "triggered on Scheduled Trial."
  • Timing: 24 hours before the class starts. The class datetime lives on the Deal as a custom property (exact name finalized with spec 02's property list).
  • Classification: Transactional. Tied to a specific confirmed booking.
  • Recipient: The Deal's associated Contact (the parent).

Decide at kickoff

Delay mechanic

HubSpot supports "wait until a date property minus N hours." The delay targets the Deal's class-datetime property minus 24 hours. If the trial is confirmed less than 24 hours before the class (same-day confirmation), the delay resolves to the past — the workflow should send immediately. Exact delay-step configuration is finalized when Ibtissam builds the workflow.

Guard against sending on cancelled / denied Deals

Between enrollment and send time, the Deal might move to Denied (staff action) or otherwise out of Scheduled Trial. The workflow needs an if/then branch right before the email action: "is the Deal still in Scheduled Trial?" If no, end. This prevents reminders firing on cancelled bookings.

Re-enrollment on reschedule

If staff reschedules a class, the Deal's class-datetime changes but the stage stays Scheduled Trial. The workflow's re-enrollment rules need to fire on "class datetime updated," not just "stage changed to Scheduled Trial" (which wouldn't re-trigger on a reschedule). Ibtissam sets re-enrollment at kickoff.

Timezone

Class datetime should be stored as UTC ISO 8601 on the Deal. The email body renders local time via HubSpot's personalization-token date formatting. Stuart confirms the Deal-property format when he wires spec 03.

Setup steps in HubSpot (once the decisions above are settled)

  1. Marketing → Workflows → Create workflow → Based on a Deal.
  2. Enrollment trigger: pipeline is Trials, stage is Scheduled Trial.
  3. Configure re-enrollment per the decision above.
  4. Delay: wait until the Deal's class-datetime property minus 1 day.
  5. If/then: Deal stage is Scheduled Trial. True → send email. False → end.
  6. Pick the 24-hour reminder template from spec 08.
  7. Turn the workflow on.

Verification plan

  1. Create a test Deal manually with class-datetime set to "now + 25 hours."
  2. Move it to Scheduled Trial. Confirm the workflow enrolled, delay shows ~23 hours pending.
  3. Edit class-datetime to "now + 5 minutes" to fast-forward. Confirm the delay recalculates.
  4. After the delay clears, check the workflow history: branch evaluated true, email sent.
  5. On a second test, move the Deal to Denied before the delay clears. Confirm the false branch ends the workflow without sending.
