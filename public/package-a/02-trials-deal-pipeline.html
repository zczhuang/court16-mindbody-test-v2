Trials Deal pipeline

Stands up a new HubSpot Deal pipeline so every booking becomes a tracked Deal rather than only a Contact property update.

What
A new Deal pipeline with the five stages the SOW named: Requested Trial · Scheduled Trial · Attended · No-Show · Denied.
Why
Today booking state lives only as Contact properties. A Deal per booking gives HubSpot workflows a first-class object to branch on and supports parents booking multiple trials over time (the current Contact-overwrite pattern doesn't).
Who
Ibtissam creates the pipeline in HubSpot (Settings → Objects → Deals → Pipelines → Create). Stuart writes Deals into it via specs 03–05.
Status

What's defined

  • The five stage names and their open / closed-won / closed-lost classification, per the SOW.
  • The Deal pipeline is the single source of truth for booking state once it's live; Contact properties continue to carry parent-identity data.
  • Every Deal is associated to exactly one Contact (the parent) via standard HubSpot Deal→Contact association.
  • The correlation ID that the booking app already generates per submission is the join key between Deal and Contact (both carry court16_correlation_id; the Contact property is defined in spec 01).

Decide at kickoff

The items below are called out as decisions for Day 1 of kickoff (May 7). Stuart brings proposals; Ibtissam + Anthony sign off.

Deal custom properties

Which Deal-level custom properties we create beyond court16_correlation_id. Candidates informed by the existing Contact schema (spec 01):

  • court16_class_id, court16_location_slug, court16_class_datetime — so pipeline views and workflows don't have to hop through the Contact for common reporting.
  • court16_denial_reason — populated when the Deal moves to Denied. Values depend on spec 05's picklist decision.

Final list + exact internal names get agreed at kickoff.

Stage probabilities + close-date conventions

HubSpot pipelines require a probability (%) + close-date behavior per stage. Industry defaults for Trials-style pipelines typically put Requested Trial in the 30–50% band and Scheduled Trial in the 70–85% band, but the exact numbers are Ibtissam's call.

Automation triggers per stage

Which workflows fire on which stage transitions is specified in:

  • Spec 04 (Requested Trial → Scheduled Trial)
  • Spec 05 (→ Denied)
  • Spec 06 (Requested Trial enrollment)
  • Spec 07 (Scheduled Trial + 24h)

Deal amount

Kids trials are free, so amount = $0. Adult intros carry the offer price. How we handle the Track-1-vs-Track-2 split (this package only delivers the kids pipeline; adult intros stay on the existing Contact-only path until a follow-on package) is agreed at kickoff.
